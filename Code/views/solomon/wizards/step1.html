Please setup your SIP domain(FQDN: Full Qualified Domain Name, i.e. mypbx.com) here. After the domain setup, the extension can make and receive calls with the domain. For example, call to extension 100 by: sip:100@mypbx.com

The domain does not requires resolvable. You can also use PBX IP address to instead of it.

SIP communication commonly uses UDP/TCP on port 5060/5061 for nonencrypted SIP sessions and the port 5063 for SIP sessions encrypted with TLS(Transport Layer Security).

PortSIP PBX supports many network transports for the SIP session, please choose one in below.

You can also add more transport such as TLS, WS, WSS in management console after completed Setup Wizard.
